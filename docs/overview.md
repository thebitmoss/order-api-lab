# システム概要

このシステムは、商品在庫を見ながら注文を受け付けるための小さな API です。

今は学習用の最小構成なので、1つの注文で扱える商品は1種類だけです。複数商品をまとめて買う形や、支払い、配送、会員情報などはまだ入れていません。

## 何ができるか

- 商品を登録する
- 商品一覧を見る
- 注文を作る
- 注文一覧を見る
- 注文をキャンセルする

## 注文と在庫の流れ

### 1. 商品を登録する

最初に、売る対象の商品を登録します。

商品には、名前、価格、在庫数があります。

### 2. 注文を作る

注文時には、どの商品を何個ほしいかを指定します。

システムは次の順で確認します。

1. 入力内容が正しいか
2. 指定された商品が存在するか
3. 在庫が足りているか

問題がなければ注文を作り、その分だけ在庫を減らします。

## 3. 在庫が足りない場合

在庫より多い数量で注文しようとした場合、その注文は受け付けません。

このとき、注文だけ作られて在庫が減らない、あるいは在庫だけ減って注文が残らない、といった中途半端な状態を避けるようにしています。

## 4. 注文をキャンセルする

注文をキャンセルすると、その注文は削除されます。

同時に、その注文で使っていた数量分だけ商品在庫を戻します。

## 運用の土台

各リクエストには requestId が付きます。

これにより、あとで障害や問い合わせを調べるときに、同じリクエストをログとエラーレスポンスの両方で追いやすくなります。

ログは method、path、status、durationMs、requestId を含む構造化形式で出すので、将来的に CloudWatch などへ流しやすい形です。
